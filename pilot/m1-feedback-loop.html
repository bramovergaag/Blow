CLASSIFYKeyword-tabel

Alle replies in de groep tussen review_sent_at en now worden samengevoegd, lowercased, en op deze keywords gematcht.

Categorie Keywords (NL + EN) Triggert Sheet-update
skip skip · stop · nee · reject Hoogste precedence — wint van alles anders status = needs_revision
edit edit · aanpas · wijzig · verander Mens past zelf aan in de sheet, daarna handmatig naar approved status = draft_ready_for_review (deadline cleared)
positive 👍 · ok · oke · prima · top · goed · ja · akkoord · approved Een van Toine of Bram bevestigt expliciet status = approved (scenario 03 pikt op)
silent Geen replies in venster Default na 2u stilte — vertrouwen in pipeline status = approved (resolution = silent_auto_approve)

DECIDEBeslissings-tak in 06b

Eén SetVariables-module met geneste if/else. Bewust simpel — geen routers nodig.

newStatus = if(hasSkip,    "needs_revision",
            if(hasEdit,    "draft_ready_for_review",
                           "approved"))                  // covers positive + silent

1 · skip gedetecteerd

Toine of Bram heeft "skip"/"stop" getypt. Pipeline stopt voor deze rij.

→ needs_revision

2 · edit gedetecteerd

Mens wil zelf aanpassen. Status blijft draft, deadline gewist zodat 06a niet re-trigger't.

→ draft_ready_for_review

3 · positive reaction

👍 / ok / ja / etc. — expliciete goedkeuring binnen het venster.

→ approved → S03 publish

4 · silent (geen replies)

Niemand heeft gereageerd in 2u. Default = vertrouwen, want de AI-output was al door scenario 02 gevalideerd op JSON-schema.

→ approved → S03 publish

FALLBACKWat als er iets mis gaat

Externe afhankelijkheden: 2chat (WA gateway), Google Sheets API, MAKE-runners. Per failure-mode is er een mitigatie.

2chat is down (4xx/5xx op send)

Make retry'et tot 3× (maxErrors=3). Als alle 3 falen: rij blijft op draft_ready_for_review zonder deadline.

→ 06a fired niet opnieuw, want filter is ook "deadline empty"; manual fallback: Bram zet handmatig op approved.

WA-groep gemute / Toine afk

Niemand ziet de notificatie. Reacties = 0.

→ Default: silent_auto_approve = approved. Geaccepteerd risico: vertrouwen op scenario 02 output. Mitigatie: maandelijks audit (% silent vs. positive).

2chat get-messages mist replies

Bijv. iemand reageert in een sub-thread die 2chat niet retourneert.

→ Risico klein omdat we from_date=review_sent_at gebruiken (geen filtering op reply-to). Edge case: gebruik group directly, geen quoted replies.

Sheet API quota / rate limit

Vooral bij 06b sweep met veel pending rijen.

→ Make backoff doet z'n werk; 06b draait elke 30 min, dus volgende run pakt achterstand op. Limiet sheet limit=10 op 06a watcher.

False-positive op keyword "ja"

Iemand typt "jaarlijks moet dit nog…" → match op "ja".

→ In code prefix " ja" met spatie in de match-string (zie blueprint). Als blijkt te vaak fout: stap over naar regex-word-boundaries of vraag explicietere reacties.

Dubbele publicatie

Wat als 06b twee keer beslist op dezelfde rij?

→ Scenario 03 heeft eigen filter status=approved AND post_id leeg. Idempotent. Tweede run van 06b vindt rij niet meer (deadline cleared).
